# AI Spending Coach — 90s demo script

Two months of categorized receipts plus sample budgets that intentionally trip
warnings on **Coffee** and **Alcohol**. Lets the Insights tab shine without
burning OCR tokens.

## Setup (one-time per fresh DB)

```bash
php artisan migrate
php artisan db:seed --class=InsightsDemoSeeder
```

Re-running the seeder is idempotent for budgets (uses `updateOrCreate`), but it
will append more receipts. Wipe with `php artisan migrate:fresh` if you need a
clean baseline.

## 90-second demo flow

1. **0:00** Open `http://127.0.0.1:8000`. Tap the **Insights** tab.
2. **0:05** "This month" card shows the running total + month-over-month delta.
3. **0:15** "Spend by category" donut highlights Groceries / Alcohol / Coffee /
   Food out as the top categories (colors match the in-app palette).
4. **0:25** AI insight cards (purple sparkles) call out the biggest mover. If
   `ANTHROPIC_KEY` is set, narratives are generated by Claude; otherwise we use
   a deterministic template fallback so the demo never breaks.
5. **0:40** Budget list shows a **warning** chip on Coffee and an **over** chip
   on Alcohol. The progress bar turns orange / red.
6. **0:55** Tap a budget row → bottom sheet opens; bump the limit with the
   slider, save, watch the bar redraw immediately.
7. **1:10** Switch to the **Tally** tab and scan a fresh receipt. Items are
   auto-categorized by Claude using the fixed taxonomy (`SpendingCategories`).
8. **1:20** As soon as the scan returns, a toast ("Coffee budget warning")
   slides in from the top — same logic powers the Insights tab.
9. **1:30** Confirm the split. The done screen polls bunq every 4s and flips
   each contact's chip from `PENDING` → `PAID` in real time.

## Where the data lives

- `receipt_items.category` (text) + `category_confidence` (decimal 0–1)
- `category_budgets` (user_id, category, monthly_limit, currency)
- All categorization happens server-side in `ClaudeController::scan` using the
  fixed `App\Support\SpendingCategories` taxonomy — anything Claude returns
  outside the list is normalized to `other`.
